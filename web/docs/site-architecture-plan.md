# EncarKorea — Архитектура сайта v2

## Результаты маркетингового исследования

### Ключевые инсайты

1. **CTA "Узнать цену" конвертит на 30-50% лучше** чем "Оставить заявку" — низкое обязательство, даёт ценность
2. **Калькулятор растаможки — главный конверсионный инструмент** в нише пригона авто
3. **Trust-блоки отсутствуют** — для бизнеса по пригону доверие = ВСЁ
4. **RUB цена должна быть основной**, KRW вторичная (пользователь думает в рублях)
5. **Страницы марок — SEO-золото**: "/catalog/hyundai/" с уникальным контентом, FAQ, ценами
6. **Мессенджер-кнопка (Telegram/WhatsApp)** — обязательна для авто-ниши в РФ
7. **Пошаговый процесс** — главный дифференциатор от агрегаторов

## Данные: что подтягиваем из API

### Текущие поля (уже есть):
- id, manufacturer, model, badge, badgeDetail
- year, formYear, priceMan/KRW/USD/RUB
- mileageKm, fuelType, transmission
- officeCity, officeName, dealerName

### Нужно добавить (расширить API клиент):
- **Displacement (объём двигателя)** — критично для калькулятора пошлин
- **Color** — уже возвращается, но пустой в текущем парсинге
- **Body type** — седан/кроссовер/хэтчбек
- **Drive type** — 2WD/4WD (парсится из badge)
- **Photo count** — для бейджа на карточке

### Для SEO-страниц марок — агрегация:
- Кол-во авто по марке/модели
- Мин/макс цена
- Популярные модели (top-5 по count)
- Средняя цена, средний пробег

## Контент-стратегия (уникализация)

### Для страниц марок:
1. **Динамические данные**: кол-во авто, ценовой диапазон, средние показатели
2. **Шаблонные тексты с вариативностью**: для каждой марки свой набор фактов
3. **FAQ**: 5-7 вопросов уникальных для марки
4. **Сравнение цен**: "в Корее от X, в России от Y, экономия до Z%"

### Для страниц моделей:
1. **Характеристики модели**: поколения, двигатели, комплектации
2. **Динамическая выборка**: актуальные авто этой модели
3. **FAQ**: специфичные для модели вопросы
4. **SEO-текст**: обзор модели, на что обратить внимание

### Для детальных страниц:
1. **Авто-генерированное описание**: из данных API (марка, модель, год, пробег, двигатель)
2. **Калькулятор**: предзаполнен данными авто
3. **Блок "Почему этот авто"**: Low mileage? New model? Good condition? — динамические бейджи

## Технические решения

### ISR (Incremental Static Regeneration):
- Главная: revalidate 10 min (как сейчас)
- Каталог: revalidate 10 min
- Страницы марок: revalidate 30 min
- Детальные: revalidate 60 min (или on-demand)

### Sitemap:
- Динамический sitemap.xml
- Все страницы марок + моделей + детальные

### Robots.txt:
- Allow all /catalog/ pages
- Disallow /api/

### Performance:
- Image optimization через Next.js Image (ci.encar.com)
- Lazy loading карточек
- Preload critical CSS + fonts
